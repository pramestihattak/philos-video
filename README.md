# philos-video

A self-hosted video streaming API server written in Go. Supports chunked upload, server-side transcoding to adaptive-bitrate HLS, live RTMP ingest with real-time multi-quality HLS delivery, Google OAuth authentication, JWT-secured playback, and Prometheus metrics.

The backend is a **pure API server** ‚Äî no HTML templates. The frontend lives in a separate repository and communicates via the OpenAPI-defined REST API.

> **Full documentation:** [`docs/README.md`](docs/README.md)

---

## Features

- **OpenAPI-first** ‚Äî all 30 endpoints defined in `definition/api.yaml`; types and routing generated by `oapi-codegen`
- **Chunked upload** ‚Äî browser-side file splitting, sequential chunks, auto-assembles on last chunk
- **VOD transcoding** ‚Äî FFmpeg pipeline: probe ‚Üí encode (720p/480p/360p) ‚Üí fMP4 HLS segments ‚Üí master playlist
- **Live streaming** ‚Äî RTMP ingest (port 1935), real-time multi-quality HLS with 2-second segments, auto-VOD on stream end
- **Google OAuth** ‚Äî cookie-based user sessions, per-user upload quota
- **JWT playback auth** ‚Äî signed tokens bound to specific video or stream IDs, passed as `?token=` query param
- **Telemetry pipeline** ‚Äî client-side event collection (TTFF, rebuffer, quality switches)
- **Prometheus metrics** ‚Äî HTTP, upload, transcode, live, delivery, and playback event counters
- **Health checks** ‚Äî `/health` (liveness) and `/health/ready` (postgres, ffmpeg, disk, RTMP port)
- **Graceful shutdown** ‚Äî worker drain, live stream `#EXT-X-ENDLIST`, 60s transcode timeout

## Quick Start

```bash
# Requirements: Go 1.22+, FFmpeg, Docker

cp .env.example .env   # fill in JWT_SECRET, GOOGLE_CLIENT_ID, etc.
make db                # start PostgreSQL in Docker
make serve             # start HTTP (:8080) + RTMP (:1935) server
```

| Endpoint | Purpose |
|----------|---------|
| `GET /health` | Liveness probe |
| `GET /health/ready` | Readiness probe (all deps) |
| `GET /api/v1/videos` | List videos |
| `GET /auth/google/login` | Start Google OAuth flow |
| `GET /metrics` | Prometheus scrape (requires login) |

Migrations run automatically on startup ‚Äî no manual SQL needed.

### Component layers

| Layer | Package | Purpose |
|-------|---------|---------|
| Entry point | `cmd/server` | Wire config ‚Üí DB ‚Üí repos ‚Üí services ‚Üí chi router |
| API handlers | `internal/server` | Implements `api.ServerInterface` (one file per handler method) |
| Generated API | `gen/api` | Types, `ServerInterface`, `HandlerFromMux` (do not edit) |
| Services | `internal/service` | Business logic (no HTTP, no SQL) |
| Repositories | `internal/repository` | SQL queries |
| Live streaming | `internal/live` | RTMP server + per-stream FFmpeg sessions |
| VOD transcoding | `internal/transcoder` | FFmpeg/FFprobe wrappers |
| Metrics | `internal/metrics` | Prometheus metric definitions + system collector |
| Health | `internal/health` | Liveness and readiness probes |
| Watchdog | `internal/watchdog` | FFmpeg process monitor + stuck-job recovery |

## Configuration

| Variable | Default | Description |
|---|---|---|
| `PORT` | `8080` | HTTP port |
| `DATABASE_URL` | `postgres://philos:philos@localhost:5433/philos_video?sslmode=disable` | PostgreSQL DSN |
| `DATA_DIR` | `./data` | Video storage root |

| `JWT_EXPIRY` | `1h` | Playback token lifetime |
| `RTMP_PORT` | `1935` | RTMP ingest port |
| `CORS_ORIGINS` | `*` | Comma-separated allowed origins for the frontend |

## Live Streaming (OBS)

1. Sign in and visit the stream keys API (`POST /api/v1/stream-keys`) to create a key (`sk_‚Ä¶`)
2. In OBS: **Settings ‚Üí Stream ‚Üí Service: Custom**
   - Server: `rtmp://localhost:1935/live`
   - Stream Key: the `sk_‚Ä¶` value
3. Start streaming ‚Äî viewers create a session via `POST /api/v1/live/{stream_id}/sessions` to get a JWT and HLS manifest URL
4. On stream end, a VOD recording is automatically created

## Encoding Ladder

| Profile | Resolution | Video | Audio | Format |
|---------|-----------|-------|-------|--------|
| 720p | 1280√ó720 | 2500 kbps | 128 kbps | HLS fMP4 (VOD) / TS (live) |
| 480p | 854√ó480 | 1000 kbps | 96 kbps | HLS fMP4 (VOD) / TS (live) |
| 360p | 640√ó360 | 400 kbps | 64 kbps | HLS fMP4 (VOD) / TS (live) |

Profiles exceeding source resolution are skipped automatically.
